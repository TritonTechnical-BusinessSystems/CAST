# Template — Release Notes (user-facing)

Generated at every General Release (MAJOR/MINOR), aggregating all changelog entries since the previous General Release. Curated, friendly, illustrated — the human story, not the raw changelog. Spec: `../conventions/changelog-and-releases.md`.

```markdown
# What's New in vMAJOR.MINOR — <Month YYYY>

<One-sentence headline of the release's theme.>

## Highlights
- **<Feature, in user language>** — what it does for you and why it matters. <illustration/screenshot>

## Improvements
- <Smaller enhancements, plainly stated>

## Fixes
- <Notable fixes users will feel — no internal jargon>

## For the power users
- <Advanced/depth items: new hooks, templates, configuration surface, etc.>

---
_Full technical detail: see CHANGELOG.md (builds <first> … <last>)._
```
